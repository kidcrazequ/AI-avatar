# 数据可视化技能 · draw-chart

> **级别**：[■■] 进阶

## 技能说明

当用户的问题涉及数据可视化时，用 ECharts 输出一份高级感图表，直接内联在回答中渲染。
这个技能只负责"画图"，不负责"找数据"——找数据是 `search_knowledge` 或 `chart-from-knowledge` 的职责。

## 触发条件

在以下场景应读取并遵循本技能：

- 用户显式要求：「用图表 / 画个图 / 可视化 / 做一张趋势图 / 做饼图 / 柱状图对比」
- 用户的问题天然适合图表表达：月度/季度/年度趋势、占比分布、同比/环比对比、KPI 看板

**触发关键词**：`图表` `可视化` `趋势` `对比` `占比` `分布` `同比` `环比` `柱状` `折线` `饼图` `画图`

## 输入

| 参数 | 是否必须 | 说明 | 示例 |
|------|---------|------|------|
| 数据 | 必须 | 要可视化的数据；优先从已检索的上下文取，否则先 `search_knowledge` 再用 | 月度销售额 |
| 图表类型 | 可选 | **用户指定时必须严格遵从**（说"折线图"就必须用 line，说"柱状图"就必须用 bar）；未指定时按数据性质选（趋势→line，占比→pie，对比→bar） | 折线 / 柱状 / 饼 |
| 标题 | 必须 | 你必须生成一个含义明确的标题 | "2024 年月度销售额" |

## 数据来源规则（**硬约束**）

按优先级：

1. **Excel / CSV 数据 → 必须用 `query_excel`** — 看 system prompt 顶部"可查询 Excel 数据源"列表，找到对应的 file 和 sheet，用 filter 精确过滤行。**禁止**用 search_knowledge 查 Excel 数据（模糊匹配会丢行）。示例：用户问"215 机型 2026 年 1~3 月设备侧效率"：
   ```
   query_excel({
     file: "00_工商储-产品质量指标dashboard_260303",
     sheet: "月度",
     filter: {
       "机型": "215",
       "月份": {"$gte": "2026-01", "$lte": "2026-03"}
     },
     columns: ["月份", "设备侧效率"]
   })
   ```
2. **非 Excel 的知识库数据** → 用 `search_knowledge` 模糊检索
3. 若用户在消息里直接贴了数据 → 使用用户数据
4. 若以上都没有 → **不要编造**。诚实说明缺什么数据，请求用户提供或指出 Excel 文件

**严禁行为**：
- ❌ 凭记忆/推测填入数字（模型训练数据不是事实）
- ❌ 用「大约」「假设」「估计」等词 + 编造的数值
- ❌ 用占位数据画出"示意图"——即使用户没明说，也要等真实数据
- ❌ **对 Excel 用 search_knowledge** 而不是 query_excel

## 执行流程

1. **确认数据来源** — 上下文有数据 / 调用检索 / 用户已提供 —— 三选一
2. **选择图表类型** — 按数据性质：趋势→line / 占比→pie / 对比→bar / 分布→scatter
3. **构造 ECharts option** — 使用下面的"输出格式硬约束"
4. **输出** — 在回答中先写一两句解读（"数据显示 X 月销售额最高..."），然后立即紧跟一个 `chart` 代码块
5. **结束** — 不要在图表后追加"希望这对您有帮助"等套话

### 工具调用纪律（**极其重要**）

- **最多 2 次 query_excel** — 第 1 次查主数据，第 2 次补充（如查对比期）。超过 2 次说明你的 filter 写错了，停下来用已有数据画图
- **有数据就画** — 只有 1 个月的数据？画 1 个点的图。只有 3 行？画 3 个点的图。**不要因为数据少就继续搜索更多数据**，用户要的是"用现有数据画图"，不是"帮我找更多数据"
- **load_skill 只调 1 次** — 加载完 draw-chart 技能后立即画图，不要再加载其他技能
- **整个流程不超过 4 轮工具调用** — query_excel(1-2次) + load_skill(1次) + 最终输出 = 最多 4 轮。如果你发现自己在第 5 轮还没画图，**立即用已有数据输出 chart**

## 输出格式硬约束

必须输出**恰好一个** fenced code block，语言标签是 `chart`，内容是合法的 ECharts option JSON。

```chart
{
  "title": { "text": "标题", "subtext": "副标题" },
  "tooltip": { "trigger": "axis" },
  "legend": { "data": ["系列名"] },
  "xAxis": { "type": "category", "data": ["1月", "2月", "3月"] },
  "yAxis": { "type": "value" },
  "series": [
    { "name": "系列名", "type": "line", "data": [120, 132, 101] }
  ]
}
```

**代码块内禁止的内容**：
- 非 JSON（注释、尾逗号、单引号、变量、**JavaScript 函数**）
- **严禁 `function` 关键字**：`formatter`、`valueFormatter`、`color` 等字段不得使用 `function()`。只能用 ECharts 的字符串模板（如 `"{b}: {c}"`）或直接省略
- 解释文字（必须放在代码块**之前**作为普通 markdown 段落）
- 空对象或空 series 数组

## UED 设计规范（摘自 ued-expert.md）

### ✅ 必须遵守

1. **禁止 3D** — `series[].type` 不得是 `bar3D` / `line3D` / `surface` / `scatter3D`
2. **Y 轴从 0 开始** — `yAxis` 不写 `min` 字段（默认 0），除非图表类型是 scatter / candlestick
3. **标题 + 副标题必填** — `title.text` 和 `title.subtext` 都要有；subtext 用来标注数据范围或单位
4. **系列数 ≤ 5** — 主题 palette 只提供 5 色，超过会重复
5. **不得硬编码颜色** — 不写 `color` 字段，让 pixel 主题自动接管
6. **X 轴类目 > 12 时改用横向** — 如月份可以竖着，但站点、SKU 这种多条目必须用横向柱状图（`yAxis.type: 'category'`, `xAxis.type: 'value'`, `series[].type: 'bar'`）
7. **单值 KPI 必须带对比** — 如果只有一个数字，必须同时展示"上期/去年同期"作为第二个系列或副标题

### ✅ 高级视觉（主题已内置，你只需配合）

主题已内置以下视觉效果，**不需要你在 JSON 里手动写**：
- 折线图自动带渐变面积填充（areaStyle）
- 柱状图自动圆角顶部
- Tooltip 自动毛玻璃效果
- 配色自动 5 色暗底优化

**你需要配合的**：
1. **折线图少写 symbol** — 不要写 `showSymbol: true`，主题默认 hover 才显示点，更干净
2. **多系列折线图** — 不要写 `areaStyle`（主题默认只对第一系列适用渐变，多系列会重叠变脏，此时省略即可）
3. **饼图用环形** — `radius: ["45%", "70%"]`，比实心饼更现代
4. **markPoint 标注极值** — 趋势图加 `markPoint: { data: [{ type: "max" }, { type: "min" }] }` 突出关键数据
5. **grid 留白** — 不要写 grid 字段，主题默认 `{top: 72, left: 56}` 已为标题和 Y 轴留够空间

### ❌ 严禁

- 3D 图表（反模式）
- 彩虹调色板（5 色以上）
- 截断 Y 轴
- 超过 12 个扇区的饼图（改用横向柱状图 + Top 10）
- 在图表内重复标题文字（tooltip / label 已经能表达）
- 手动写 `areaStyle`、`itemStyle.borderRadius`、`tooltip` 样式（**主题已接管**，手写会覆盖）

## 示例

### 示例 1：月度趋势（line）

**用户问**：用图表展示 2024 年各月销售额

**检索**：从 `月度销售.xlsx` sheet `2024` 得到 12 条数据

**你的回答**：

2024 年销售趋势显示上半年增长平稳，6 月达到年度峰值，下半年略有回落。全年总销售额 1,485 万元。

```chart
{
  "title": { "text": "2024 年月度销售额", "subtext": "单位：万元 | 数据来源：月度销售.xlsx" },
  "tooltip": { "trigger": "axis" },
  "xAxis": {
    "type": "category",
    "data": ["1月","2月","3月","4月","5月","6月","7月","8月","9月","10月","11月","12月"]
  },
  "yAxis": { "type": "value", "name": "万元" },
  "series": [
    {
      "name": "销售额",
      "type": "line",
      "data": [95, 108, 115, 122, 130, 148, 135, 128, 120, 115, 112, 157],
      "markPoint": { "data": [{ "type": "max", "name": "最高" }] }
    }
  ]
}
```

### 示例 2：品类占比（pie）

**用户问**：画个图看下各品类销售占比

**检索**：从 `品类销售.xlsx` 得到 5 条数据

**你的回答**：

全年销售中储能系统占比最高（42%），其次是充电桩（28%）。

```chart
{
  "title": { "text": "2024 年品类销售占比", "subtext": "单位：% | 数据来源：品类销售.xlsx" },
  "tooltip": { "trigger": "item", "formatter": "{b}: {c} ({d}%)" },
  "legend": { "orient": "vertical", "left": "right" },
  "series": [
    {
      "name": "品类占比",
      "type": "pie",
      "radius": ["40%", "70%"],
      "data": [
        { "name": "储能系统", "value": 42 },
        { "name": "充电桩", "value": 28 },
        { "name": "光伏组件", "value": 15 },
        { "name": "BMS", "value": 10 },
        { "name": "其他", "value": 5 }
      ]
    }
  ]
}
```

### 示例 3：横向柱状对比（多条目）

**用户问**：15 个站点今年业绩对比

**检索**：15 条数据

**你的回答**：

华东区的 3 个站点表现最好，华北区整体偏弱。

```chart
{
  "title": { "text": "2024 年 15 站点销售额对比", "subtext": "单位：万元" },
  "tooltip": { "trigger": "axis" },
  "grid": { "left": 120 },
  "xAxis": { "type": "value", "name": "万元" },
  "yAxis": {
    "type": "category",
    "data": ["华东-01","华东-02","华东-03","华东-04","华北-01","华北-02","华北-03","华南-01","华南-02","华南-03","华中-01","华中-02","西北-01","西南-01","西南-02"]
  },
  "series": [
    {
      "name": "销售额",
      "type": "bar",
      "data": [185, 172, 168, 145, 98, 92, 88, 155, 148, 142, 132, 125, 112, 128, 119]
    }
  ]
}
```

## 失败回退

如果数据不足以画图，**不要输出空图表**。写一段清晰的说明：

> 无法生成图表：知识库中没有 2023 年的月度销售数据。请先通过 IMPORT 导入 Excel / CSV 文件，或直接在消息里告诉我具体数据（例如：1 月 100 万，2 月 120 万...）。

## 禁止事项

- ❌ 画图前不核实数据来源
- ❌ 输出多个 ```chart 代码块（一次一个）
- ❌ 把解读文字写进 JSON 里作为 subtext
- ❌ 给 series 加 color 字段（主题接管）
- ❌ 使用 markLine/markPoint 时引用不存在的数据索引
