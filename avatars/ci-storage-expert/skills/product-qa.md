# 产品问答

> **级别**：🟢 基础
> **版本**：v1.0

---

## 技能说明

回答 ENS-L262 / ENS-L419 等产品相关问题时，自动检索相关章节、确保覆盖完整、标注数据来源。

## 触发条件

当用户问到以下关键词时触发：
- 产品型号（L262、L419、ENS-xxx）
- 产品属性（参数、尺寸、重量、占地、布局、接口、安装、消防、电气、温控、维护、故障……）

## 执行流程

### Step 1：识别问题类型

将用户问题归类到以下一个或多个类型：

| 问题类型 | 关键词示例 |
|---------|----------|
| **整机参数** | 参数、规格、spec、多大、多重 |
| **内部布局** | 布局、内部、结构、组成、长什么样 |
| **电芯/电池** | 电芯、电池、模块、电池簇、容量、SOC |
| **PCS变流器** | PCS、变流器、功率、效率、接口、并机 |
| **温控系统** | 温控、液冷、散热、冷却、温度 |
| **消防系统** | 消防、灭火、探测器、全氟己酮、泄爆、报警 |
| **安装要求** | 安装、场地、占地、地基、固定、运输、吊装 |
| **电气接线** | 接线、电气、接地、上电、下电、通讯、电网 |
| **维护保养** | 维护、保养、巡检、清洁、故障、报错 |
| **选型对比** | 对比、区别、选哪个、vs、选型 |

### Step 2：映射知识章节

根据问题类型，自动确定需要读取的知识文件和章节：

```
整机参数  → 产品概述 + 核心参数 + 柜体参数
内部布局  → 内部布局 + 温控系统 + 消防系统 + 柜体外观布局 + 操作开关位置
电芯/电池 → 电芯参数 + 电池模块 + 电池簇
PCS变流器 → PCS参数 + PCS前面板接口 + PCS指示灯状态
温控系统  → 温控系统 + 散热气流路径 + 内部布局（液冷机组位置）
消防系统  → 消防系统 + 内部布局（探测器/灭火装置/声光报警位置）+ 柜体外观（泄爆/水消防接口）
安装要求  → 安装要求 + 柜体参数（尺寸/占地）+ 固定方式 + 运输 + 存储 + 地基要求
电气接线  → 电气接线 + 接口规格 + 上下电顺序 + 五大安全法则 + 系统接线架构
维护保养  → 维护周期 + 常见故障诊断 + 补漆措施 + 系统清洁
选型对比  → 两个产品文件的对比表 + 各维度差异
```

**关键规则：「内部布局」类问题必须同时覆盖设备布局 + 消防布局 + 温控布局 + 电气接口位置。**

### Step 3：读取知识文件

1. 确定产品型号 → 读取对应的 `knowledge/products/envision-L2xx.md`
2. 按 Step 2 映射结果，定位到具体章节
3. 如涉及两个产品对比，两个文件都要读

### Step 4：完整性自检

回答前，对照以下清单检查（根据问题类型勾选适用项）：

**通用检查：**
- [ ] 是否回答了用户的核心问题
- [ ] 数据是否标注了来源（手册章节/图号/表号）
- [ ] 是否有知识缺口需要告知用户

**布局类问题附加检查：**
- [ ] 设备位置（从上到下、从左到右）是否完整
- [ ] 消防组件位置是否说明（探测器/灭火装置/声光报警/泄爆）
- [ ] 温控组件位置是否说明（液冷机组/液冷管道/除湿机）
- [ ] 电气接口位置是否说明（交流接线/通讯口/接地点）
- [ ] 散热气流方向是否说明

**参数类问题附加检查：**
- [ ] 数值是否带公差/精度（如电芯尺寸±0.5mm）
- [ ] 单位是否明确
- [ ] 是否区分了「手册标注值」和「工程经验估算值」

**安装类问题附加检查：**
- [ ] 是否区分了柜体本体尺寸和实际安装占地
- [ ] 单台和多台场景是否都覆盖
- [ ] 固定方式/运输方式是否说明

**对比类问题附加检查：**
- [ ] 是否指出了容易混淆的差异（如电芯/模块"看似相同实则不同"）
- [ ] 是否给出了选型建议

### Step 5：输出回答

格式要求：
- 先直接回答核心问题（不要铺垫）
- 用表格呈现结构化数据
- 用 ASCII 图呈现布局/架构/流程
- 在关键数据后标注来源：`（数据来源：用户手册 图X-X / 表X-X）`
- 如有 L262 vs L419 差异点，主动提示
