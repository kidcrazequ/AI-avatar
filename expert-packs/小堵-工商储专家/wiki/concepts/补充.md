# 补充

“补充”在本知识库中指对现有文档内容的增强性添加行为，主要用于弥合知识缺口、完善信息维度或提升数据可用性。其核心用途包括：将PDF等原始文档中的图片内容识别后结构化为文本并注入对应Markdown文件；针对手册中缺失的关键技术参数（如电气安全、PCS性能、电池系统等工程级指标），通过外部权威资料进行定向填充；以及在知识库目录结构中预留待补充位置，以支持后续扩展。该操作强调**来源可追溯**（标注`[来源: ...]`）、**格式统一**（仅接受Markdown）、**内容精准**（不推导、不猜测，仅补充明确可获取的信息）。

| 属性 | 说明 |
|------|------|
| **触发场景** | PDF图片内容识别、知识缺口项确认、新文档导入 |
| **执行主体** | 知识编辑人员或自动化提取工具 |
| **目标载体** | `knowledge/` 目录下的 Markdown 文件 |
| **约束条件** | 不得添加手册中未提及且无法从外部验证的信息；不得修改原始文档表述；需标注具体来源 |
| **典型动作** | 将 `images/` 中图片识别出的关键参数写入对应 `.md` 文件；在 `README.md` 中更新目录结构与补充说明 |

相关实体：知识缺口、用户手册、Markdown、图片识别、EMS、BMS、PCS
