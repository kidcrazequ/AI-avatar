# 知识

“知识”在本系统中指支撑智能体（如“小堵—工商储专家”）进行专业问答与决策的结构化、可读取、可溯源的领域信息集合。它以 Markdown 格式文件为核心载体，来源于原始技术文档（如用户手册、白皮书、作业指导书等），经人工或识别处理后纳入 `knowledge/` 目录；其内容需满足单位明确、图文互证、来源可追溯等质量标准。知识并非泛义的认知成果，而是严格限定于工程交付与现场运维所需的**可执行参数、操作步骤、安全阈值、协议细节及认证依据**——如片段所示，手册中大量缺失项（如“MPPT范围”“螺栓紧固扭矩”“Modbus寄存器地址”）正因其未被显式提供而构成“知识缺口”，反向界定了本系统中“知识”的边界：它必须是**显性、具体、可验证、可引用**的信息单元。

| 属性 | 值 | 说明 |
|------|-----|------|
| **存在形式** | Markdown 文件（`.md`） | 所有知识文件均采用 Markdown 格式，分身可直接解析 |
| **存储位置** | `knowledge/` 目录及其子目录（如 `images/`） | 包含文本文件与补充性关键页面图片（PNG） |
| **来源要求** | 明确标注原始文档名称与版本 | 如 `ENS-L262-01用户手册 -V1.pdf` |
| **内容标准** | 关键数值带单位、图片数据已转录、表格结构化 | 遵循《知识质量标准》检查项 |
| **功能定位** | 支撑分身回答、标注来源、识别缺口、驱动补全 | 缺失即触发“知识缺口”分析，并指引补充文档类型 |

相关实体：知识缺口、用户手册、运维白皮书、安装作业指导书、知识库、Markdown、小堵—工商储专家
