## ✅ Phase 7 功能范围

### 1. 测试用例管理 ✅
- **TestManager** - 后端测试管理器
- **功能**:
  - 读取 tests/cases/*.md 测试用例
  - 解析测试用例格式（frontmatter + PROMPT + RUBRICS + MUST_CONTAIN + MUST_NOT_CONTAIN）
  - 创建新测试用例
  - 删除测试用例
  - 保存测试报告
  - 获取测试报告列表

### 2. 测试运行器 ✅
- **TestRunner** - 前端测试运行服务
- **功能**:
  - 运行单个测试用例
  - 运行多个测试用例（批量）
  - 调用 DeepSeek API 获取 AI 回复
  - 超时控制（按测试用例配置）
  - 评估回复质量（MUST_CONTAIN/MUST_NOT_CONTAIN）
  - AI 评估 RUBRICS（使用 DeepSeek 评分）
  - 生成测试结果和反馈

### 3. 测试管理界面 ✅
- **TestPanel** - 测试管理面板
- **功能**:
  - 显示所有测试用例（按分类分组）
  - 选择测试用例（单选/全选）
  - 运行测试（显示进度）
  - 显示测试结果（通过/失败统计）
  - 查看详细结果（评估反馈、AI 回复）
  - 保存测试报告

### 4. 集成到主界面 ✅
- **App.tsx 集成**
- **功能**:
  - 顶部工具栏添加"自检"按钮
  - 点击打开测试面板
  - 传递 avatarId、apiKey、systemPrompt

## 🚀 如何测试

### 测试场景 1: 查看测试用例
1. 启动应用：`npm run dev`
2. 点击顶部的"自检"按钮
3. 查看测试用例列表（按分类分组）
4. 验证显示 45 个测试用例

### 测试场景 2: 运行测试
1. 在测试面板中选择测试用例（默认全选）
2. 点击"运行测试"按钮
3. 观察进度显示（进度条、状态消息）
4. 等待测试完成（约 5-10 分钟，取决于测试用例数量）
5. 查看测试结果（通过/失败统计）

### 测试场景 3: 查看详细结果
1. 在测试结果列表中点击一个测试用例
2. 查看右侧详细信息
3. 验证显示：评估反馈、AI 回复、得分、耗时
4. 点击其他测试用例查看不同结果

### 测试场景 4: 测试报告保存
1. 运行测试完成后
2. 检查 `avatars/ci-storage-expert/tests/reports/` 目录
3. 验证生成了新的测试报告文件（report-{timestamp}.json）
4. 打开文件查看报告内容
