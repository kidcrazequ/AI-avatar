目的

为规范远景能源产品的设计安全（以下简称DFS）管理，保证产品设计开发符合安全标准，提升产品本质安全和竞争力，特制定本标准。

范围

本标准适用于远景能源所有按照EDV流程开发的产品、部件、工装和工艺的DFS管理。

本标准适用于产品开发中直接影响人身安全、健康和环境等保护措施的本质安全设计。

术语

专业术语

定义

DFS

Design for Safety设计安全

设计安全范围包括人身安全、健康和环境保护维度。

本质安全设计措施

GB/T 15706.1中定义：本质安全设计措施是指通过适当选择机器的设计特性和/或明确人员与机器的交互作用来消除风险或减小风险。如几何因素、材料选择、防火防爆、物理特性、通用技术知识、稳定性、维修性、人类工效学原则遵循、电气维修、气动液压危险、控制系统本质安全等。

安全强相关

特性

为避免产品可能对人身安全、健康、环境以及产品本身带来危害的产品特性，包括人身安全、健康、环境和产品本身的安全，如安全锚点及挂点承载力，逃生救援通道畅通性，自动消防灭火效率，电气部件安全操作功能等。

安全装置

用于提供保护的物理屏障，消除或减少危险因素，设计为产品的组成部分或附属装置，常见的安全装置有联锁装置，双手操作式装置，自动停机装置，限位装置，锁定装置，隔离装置，防坠落装置，绝缘接地装置等。

安全物料

为保证产品本身的安全及人身安全配置的物资，如消防系统，灭火器，急救物品，逃生、救援装置，安全标识等。

职责

产品线CE

产品线CE是产品DFS第一责任人，应指定/任命产品线DFS SE负责统筹产品DFS管理工作，持续提升产品的安全可靠性；

推动各部门指定/任命DFS代表，对接产品线DFS SE，落实DFS工作要求；

建立并维护DFS SE和DFS代表人员清单，当发生变更时应通过公告邮件通知到相关业务和行管部门。

DFS SE

负责对接EHS保证部，制定DFS工作计划，跟踪产品线DFS KPI和行动项闭环落地；

负责推动EDV流程中安全赛道的DFS工作要求和交付物/件的内嵌落地，并监督TBB/CBB按验收标准及时交付；

负责策划产品线DFS内建组织架构，推进DFS代表和TBB/CBB CE/SE的赋能培训和内查内纠，并监督产品线各角色落实DFS职责；

依据法律法规、规范标准和失效案例制定《产品安全设计导则》,经领域专委会评审修订后发布，并持续跟踪产品安全法规和行业最佳实践技术，定期更新和完善；

负责推动或主导因产品失效导致的安全事故/事件的回溯或调查，并及时总结经验优化DFS细则，提升产品本质安全；

依据17.3.1 EHS审核管理流程开展产品DFS审核，输出产品DFS符合度报告。

DFS 代表

负责对接DFS SE，定期回顾和填报DFS考核指标，推动和监督部门DFS工作落地；

推动和监督本部门CE/SE按照DFS内嵌要求和验收标准及时输出交付物/件；

负责组织开展DFS符合度的内查内纠和持续改善，提升TBB/CBB的安全性。

TBB/CBB CE/SE（含工艺维度）

TBB/CBB CE/SE是所负责产品的DFS具体责任人，负责制定和定期更新DFS细则及DFS Checklist，经领域专委会评审后发布；

负责制定包含安全维度的Golden版风险识别和评估清单，明确EHS KPC/KCC，并将风险控制措施通过工艺规范和工艺文件承载和传递；

依据DFS细则和DFS Checklist,开展TBB/CBB的自查自纠,并制定整改和优化措施；

推动产品运营过程DFS问题的解决，必要时升级至相应技术领域专委会评审决策。

EHS保证部

负责制定和持续优化本标准，确保产品开发流程中安全赛道的DFS内嵌落地；

负责开展DFS SE和DFS代表的赋能培训，推动关键角色完成认证考核；

推动和协助产品线关键DFS问题的解决和专项技术攻关；

负责开展产品线DFS标准符合度审核，并组织策划业务端的DFS绩效评价。

EHS合规监察部

负责开展各产品线、各业务部门的DFS合规监察和审核后行动项追踪闭环。

基于EDV流程的DFS Golden版安全控制点

1.4 概念设计(PDTR1)

活动：设定产品开发压力，并传递DFS压力至TBB

责任角色：DFS SE

输入：产品需求、属性压力、《产品安全设计导则》

输出：《产品DFS压力耐力清单》

工作要求：

DFS SE应根据《产品安全设计导则》《产品技术规范》及历史失效，梳理出影响产品开发的关键属性要求，按照模板输出《产品DFS压力耐力清单》；

通过数字化平台(如工程管理平台)，作为产品的整体压力传递给TBB。

活动：制定产品需要测试验证的安全特性清单

责任角色：DFS SE

关联角色：TBB/CBB SE

输入：产品需求、属性压力、《产品安全设计导则》

输出：《产品需测试验证安全特性清单》

工作要求：DFS SE应协同TBB/CBB SE，梳理产品中安全要求和产品配置的安全装置或物料，明确需要做测试验证的产品属性和安全装置或物料的功能，输出《产品需测试验证安全特性清单》。

1.5 概念验证(PDTR2)

活动：检查DFS压力在各TBB的传递和分解情况

责任角色：DFS SE

输入：《产品安全设计导则》《产品DFS压力耐力清单》

输出：《产品DFS压力耐力清单》更新版(若有)

工作要求：根据产品需求和设计规范，检查《产品DFS压力耐力清单》的完备性，若发生变更，及时在工程平台更新《产品DFS压力耐力清单》。

活动：检查《产品需要测试验证安全特性清单》完整性

责任角色：DFS SE

关联角色：TBB/CBB SE

输入：《产品安全设计导则》《产品需测试验证安全特性清单》

输出：《产品需测试验证安全特性清单》更新版(若有)

工作要求：根据TR评审意见，检查《产品需测试验证安全特性清单》是否完整，若有变更，及时更新清单。

1.6 系统设计(PDTR3)

活动：识别和判断TBB压力耐力清单，承接产品分解的DFS压力

责任角色：TBB SE

输入：产品DFMEA清单、《产品DFS压力耐力清单》

输出：《产品TBB层级DFS压力耐力清单》

工作要求：

根据《产品DFS压力耐力清单》，检查TBB承接的DFS压力完整性；

TBB SE本产品TBB下的DFS压力分解和传递至CBB。

活动：识别和判断CBB压力耐力偏差清单

责任角色：CBB SE

输入：产品DFMEA清单、《产品DFS压力耐力清单》《产品TBB层级DFS压力耐力清单》

输出：《产品CBB层级DFS压力耐力清单》

工作要求：根据《产品TBB层级DFS压力耐力清单》，检查CBB承接的DFS压力完整性，并判断DFS压力是否存在偏差项。

1.7 CBB/TBB并行开发(PDTR4A) – 详细设计

活动：识别安全强相关装置/物料清单

责任角色：DFS SE

关联角色：部件开发代表、TBB/CBB SE

输入：《产品DFS压力耐力清单》《产品需测试验证安全特性清单》《产品CBB层级DFS压力耐力清单》《产品TBB层级DFS压力耐力清单》

输出：《安全强相关装置/物料清单》

工作要求：根据产品安全特性和压力耐力清单，识别安全强相关的装置和物料，输出《安全强相关装置/物料清单》。

活动：识别并输出标记安全强相关KPC/KCC清单

责任角色：CBB SE

输入：Golden DFMEA、Golden PFMEA、《CBB技术规范》

输出：《EHS KPC/KCC清单》

工作要求：

建立Golden版CBB层级产品开发风险评估(FMEA等)，作为CBB产品开发的基础；

根据产品风险识别和评估，输出安全强相关的《EHS KPC/KCC清单》。

活动：制定安全强相关装置/物料(含安全标识)的使用方法、维护及检查的工艺规范

责任角色：CBB SE

输入：《安全强相关装置/物料清单》、《产品安全设计导则》、《CBB技术规范》、《EHS KPC/KCC清单》

输出：《CBB工艺规范》

工作要求：

根据安全装置或物料的特点、场景和使用方法以及EHS KPC/KCC清单，制定符合产品特性的工艺规范，输出包含使用方法、维护及检查等内容的《CBB工艺规范》；

将《CBB工艺规范》及安全要求传递给产品工艺团队(如风机产品线的整机工艺)。

活动：制定包含安全操作要求的CBB《产品安全操作手册》

责任角色：CBB SE

输入：《安全强相关装置/物料清单》《CBB工艺规范》

输出：《产品安全操作手册》

工作要求：按照CBB工艺规范的技术条款，制定包含安全操作要求的CBB《产品安全操作手册》，并将手册传递给DFS SE。

活动：制定整个产品的《产品安全手册》

责任角色：DFS SE

输入：《安全强相关装置/物料清单》、《CBB工艺规范》、CBB层级《产品安全操作手册》

输出：《产品安全手册》

工作要求：根据安全相关CBB的工艺规范和安全操作手册的内容，组织梳理和集成整机/样机/样件的安全设计和操作要求，制定和输出整机/样机/样件的《产品安全手册》。

活动：自检CBB DFS Checklist符合度

责任角色：CBB SE

输入：《产品安全设计导则》《CBB技术规范》《CBB-DFS Checklist》

输出：《CBB层级DFS Checklist自检报告》及偏差清单

工作要求：

按照DFS细则及DFS checklist内容，逐项检查产品的符合性，完成《CBB层级DFS Checklist自检报告》；

针对自检发现的偏差项，应及时拉通DFS SE和DFS代表评审，必要时升级至领域专委会评审决策。

活动：自检CBB LOTO硬件能力符合度

责任角色：CBB SE

输入：《产品安全设计导则》《CBB技术规范》《CBB-DFS Checklist》《设备上锁挂牌硬件能力评估表》

输出：《CBB层级部件上锁挂牌能力（LOTO）评估报告》及偏差清单

工作要求：

按照《设备上锁挂牌硬件能力评估表》开展LOTO能力评估，输出《CBB层级部件上锁挂牌能力（LOTO）评估报告》，并评审输出是否存在偏差；

若需具备上锁挂牌能力，应输出上锁挂牌标牌和配置要求。

活动：评审CBB自检报告和LOTO能力评估报告的DFS偏差项

责任角色：CBB SE

关联角色：部件开发代表、DFS SE

输入：《CBB层级DFS Checklist自检报告》《CBB层级部件上锁挂牌能力（LOTO）评估报告》

输出：评审结论及偏差管理要求

工作要求：

根据CBB自评输出DFS Checklist报告和LOTO能力评估报告，确定评审结论；

针对偏差项制定行动措施和闭环要求。

1.8 样机验证和样机准备(PDTR4B)

活动：确认作业安全风险评估活动的适用性

责任角色：工艺TBB SE

关联角色：DFS SE

输入：《存量工艺》、《CBB工艺规范》、《EHS KPC/KCC清单》、Golden版《作业安全风险评估清单》

输出：评审结论

工作要求：

建立Golden版工艺维度的作业安全风险评估表作为TBB工艺开发的基础；

根据产品存量工艺的成熟性和安全风险评估清单的完整性，确定是否重新开展针对工艺的作业安全风险识别和评价。

活动：更新工艺《作业安全风险评估清单》—如Golden无变更

责任角色：工艺TBB SE

输入：《存量工艺》、《CBB工艺规范》、DFMEA、PFMEA、Golden版《作业安全风险评估清单》《EHS KPC/KCC清单》

输出：《作业安全风险评估清单》

工作要求：

若工艺步骤及作业活动未变更，提交Golden版《作业安全风险评估清单》作为交付物；

若工艺步骤及作业活动存在变更，应增加变更后的工艺活动风险辨识和评估，输出更新后的《作业安全风险评估清单》；

工艺TBB SE应将《作业安全风险评估清单》传递至执行端EHS团队，以便执行端及时掌握现场作业的安全风险。

活动：编制工艺手册

责任角色：工艺TBB SE

输入：《存量工艺》、《CBB工艺规范》、CBB《产品安全操作手册》、《产品安全手册》《作业安全风险评估清单》

输出：《TBB工艺手册》

工作要求：

按照存量技术路线和已识别的风险评估清单，明确各风险点的安全控制措施；

各维度工艺TBB (包括但不限于制造，包装、运输，安装、调试、运维、部件更换等)制定和发布包含安全控制措施的《TBB工艺手册》。

活动：评审工艺安全维度措施的完整性

责任角色：工艺TBB CE/SE

关联角色：DFS SE、PQA、CBB CE/SE、制造EHS代表、样机/样件交付EHS代表

输入：《TBB工艺手册》

输出：评审结论

工作要求：组织评审工艺文件中安全措施的完整性和合理性，输出手册的评审及发布意见。

1.9 样机功能验证(PDTR5)

活动：验证工艺安全措施的完整性（与制造、物流、交付与服务现场EHS代表）

责任角色：工艺TBB SE

关联角色：制造EHS代表、物流EHS代表、交付与服务现场EHS代表

输入：《CBB工艺规范》、《TBB工艺手册》

输出：《CBB工艺规范》、《TBB工艺手册》更新版(若有)

工作要求：组织验证工艺文件中安全措施的完整性和有效性，识别当前风险控制措施的遗漏点和优化项，输出和发布更新和优化版的《CBB工艺规范》和《CBB工艺手册》。

活动：验证安全强相关装置/物料的装配和安装完整性

责任角色：CBB SE

关联角色：DFS SE、DFS代表
