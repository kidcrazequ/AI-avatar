# Windows 下 AI 分身安装教程

适用对象：在 Windows 10 / Windows 11 64 位电脑上安装和使用 Soul AI 分身桌面端的用户。

## 一、安装前准备

1. 使用 Windows 10 / Windows 11 64 位系统。
2. 至少预留 2GB 以上磁盘空间。
3. 准备好团队提供的 Windows 安装包，文件名通常类似：

```text
AI分身-0.18.4-win-x64.exe
```

4. 准备一个可用的大模型 API Key，用于首次配置“对话模型”。
5. 如果要导入别人做好的分身，提前准备对方导出的分身包 JSON 文件，通常类似：

```text
某个分身.soul.json
```

## 二、安装 AI 分身

1. 双击团队提供的 `AI分身-版本号-win-x64.exe`。
2. 如果 Windows SmartScreen 提示“Windows 已保护你的电脑”，确认安装包来源可信后点击“更多信息” → “仍要运行”。
3. 进入安装向导后选择安装目录。普通用户使用默认目录即可。
4. 保持桌面快捷方式 / 开始菜单快捷方式勾选。
5. 点击安装，完成后从桌面快捷方式或开始菜单启动“AI分身”。

## 三、首次进入

启动后先完成两件事：

1. 安装或导入一个分身。
2. 配置对话模型。

如果首页还没有分身，可以点击“专家包”安装内置专家，也可以点击“导入分身包”导入别人发来的 `.soul.json` 分身包。

## 四、配置对话模型

进入任意分身工作台后，在顶部导航点击“更多” → “设置”，进入设置面板。

在左侧选择 `CHAT / 对话模型`，填写：

```text
API KEY   你的模型 Key
BASE URL  模型服务地址，例如 https://api.deepseek.com/v1
MODEL     模型名，例如 deepseek-chat
```

填写后点击 `TEST` 测试连接，通过后点击 `SAVE` 保存。

常见配置示例：

```text
DeepSeek:
BASE URL = https://api.deepseek.com/v1
MODEL    = deepseek-chat

OpenAI 兼容服务:
BASE URL 通常以 /v1 结尾
MODEL 按服务商提供的模型名填写

本地 Ollama:
在设置左侧选择 LOCAL / 端侧（本地）
默认 BASE URL = http://localhost:11434/v1
API KEY 可填 ollama
MODEL 填本机已 pull 的模型名
```

## 五、导入分身包

分身包是 Soul Pack JSON 文件。当前主进程默认导出的文件名是：

```text
<分身ID>.soul.json
```

这里的“分身包”不是用户手动把分身目录压缩出来的 `.zip`。`.zip` / `.7z` / `.rar` 归档导入是知识库里的“批量导入资料”能力，不是首页的“导入分身包”。

导入步骤：

1. 在首页点击“导入分身包”。
2. 点击选择文件按钮，在文件选择器中选择对方发来的 `.soul.json` 或其他 Soul Pack JSON 文件。
3. 查看预览信息，包括分身名称、版本、文件数、是否包含记忆等。
4. 如果本机已存在同名分身：
   - 选择“覆盖更新（推荐）”：更新人设、技能、知识库，保留本机记忆、模型配置、`_raw` 原始资料和本地新增文件。
   - 谨慎选择“完全重置”：会清空该分身目录后按包重写，本机记忆和导入后新增数据会被删除。
5. 点击“确认导入”或“覆盖更新”。
6. 导入完成后进入该分身，创建新对话验证。

## 六、安装内置专家包

如果没有外部分身包，可以先使用内置专家包：

1. 首页点击“专家包”。
2. 选择需要的专家，例如设计、财务、法务、项目管理等。
3. 点击“安装专家包”。
4. 安装完成后进入分身工作台。

如果专家包列表为空，说明安装包可能缺少 `expert-packs` 资源，请更换完整安装包。

## 七、开始使用

配置模型并进入分身后：

1. 点击左侧 `[+] NEW CHAT` 创建新对话。
2. 输入一句测试问题：

```text
请介绍一下你能做什么。
```

3. 如果能正常回复，说明安装、分身和模型配置都已完成。

## 八、常见问题

### 1. 双击后没有窗口

优先确认安装包版本和来源。如果仍然打不开，查看错误日志：

1. 如果应用能打开：进入“更多 → 设置 → 日志与反馈”，点击“导出错误日志到桌面”。
2. 如果应用完全打不开：在资源管理器地址栏尝试打开：

```text
%APPDATA%\soul-desktop\logs
%APPDATA%\AI分身\logs
```

日志目录通常是第一个（`soul-desktop`）。如果第一个打不开或没有内容，再试第二个。

找到 `error-日期.log` 后发给开发者排查。

### 2. 提示 API Key 或网络错误

进入“更多 → 设置 → CHAT / 对话模型”，检查：

1. API Key 是否正确。
2. BASE URL 是否按服务商要求填写。
3. MODEL 是否是服务商实际支持的模型名。
4. 公司网络是否需要代理或白名单。

修改后点击 `TEST`，通过后再 `SAVE`。

### 3. 导入分身包失败

确认文件是 Soul Pack JSON 文件，通常后缀是 `.soul.json`。不要选择普通知识库 JSON、配置 JSON 或压缩包。

如果提示文件被修改、token 过期或读取失败，重新点击“导入分身包”并重新选择文件。

### 4. 导入后回答不像预期

先确认导入的是完整分身包。如果预览或导入完成页提示缺失二进制资源，相关 PDF、Excel、图片等源文件可能不完整。

如果是同名分身更新，优先使用“覆盖更新（推荐）”；除非明确要清空本机数据，不要使用“完全重置”。

## 九、验收清单

1. 能从桌面快捷方式打开 AI 分身。
2. 至少有一个分身可进入。
3. “更多 → 设置 → CHAT / 对话模型”测试通过。
4. 能创建新对话并收到回复。
5. 如果导入了分身包，知识库和技能页能正常打开。
6. “更多 → 设置 → 日志与反馈 → 导出错误日志到桌面”能生成日志文件。
