# Soul 小猪主形象与像素精灵标准

> 版本：v0.1  
> 日期：2026-06-25  
> 用途：作为「AI 分身办公室」角色 IP、像素精灵、动作资产和专家变体的统一标准。

## 1. 方向判断

Soul 办公室不再以 Marvis 页面复刻为目标，而是转向「拟人动物像素 RPG 办公室」：

- 角色先成立，再进入办公室。
- 小猪是 Soul 的主形象，不是临时贴图。
- 办公室是 AI 分身运行状态的可视化空间，不是装饰背景。
- 风格参考方向是「拟人动物 + 像素 RPG + 办公室群像」，但不复制任何现有游戏角色、场景或制服设定。

## 2. 主形象定位

名称暂定：Soul 小猪分身

角色关键词：

- 聪明
- 可靠
- 有一点认真和执拗
- 偏 IT 男 / 产品研发同事气质
- 办公室里的 AI 工程师分身
- 可爱但不幼稚
- 像素 RPG 角色，而不是卡通 mascot 贴纸

一句话设定：

> 一只穿短袖 T 恤、工装裤和休闲鞋，挂着脖挂式耳机，在办公室里替你查资料、写方案、开会、执行任务的像素小猪 AI 工程师。

## 3. 轮廓规则

必须保留的识别点：

- 圆头，头部略大于身体。
- 短耳，左右轮廓要能看出猪耳。
- 猪鼻必须清晰，正面和 3/4 视角都能识别。
- 身体短小结实，不能拉成长人比例。
- 手脚短，动作幅度清楚。
- 站姿略向前倾，像准备开始工作。

禁止方向：

- 不做普通人类角色加猪鼻。
- 不做纯宠物猪。
- 不做过度婴儿化的大头玩偶。
- 不做西装、皮鞋、正装顾问或传统白领制服感。
- 不直接照搬《动物迷城》或其他现有游戏的角色脸型、服装和监狱叙事符号。

## 4. 基础服装

主形象默认服装：

- 宽松短袖 T 恤作为主服装。
- hoodie、外套、工装马甲只能作为专家变体或天气/场景变体，不能作为主形象默认服装。
- 宽松休闲裤或工装裤。
- 低帮休闲鞋 / 运动鞋，禁止皮鞋。
- 小工牌，挂在胸前或侧边。
- 脖挂式耳机，挂在颈部，表示随时接入对话和任务。
- 斜挎电脑包，里面可露出笔记本、键盘、文件或工具。

服装目的：

- 把「猪」从动物拉到「AI 分身工程师」。
- 为专家变体留下可换区域。
- 在小尺寸像素下也能读出 IT 同事感，而不是正式白领感。

## 5. 像素规格

推荐先做两套规格：

### A. 办公室主规格

- 单帧尺寸：`48x64 px`
- 适用：办公室走动、站立、坐下、点击选中、任务状态。
- 视角：3/4 俯视，适合等距或伪等距办公室。
- 轮廓：外描边 1px，深色轮廓优先。

### B. UI 小头像规格

- 单帧尺寸：`32x32 px`
- 适用：消息气泡、分身列表、任务面板、状态 chip。
- 只保留头部、猪鼻、耳麦、少量服装颜色。

### C. 高清展示规格

- 单帧尺寸：`128x128 px`
- 适用：首页空状态、角色详情、专家包展示。
- 不参与办公室运行，只作为宣传和选角展示。

## 6. 基础动作集

第一批动作只做工作闭环，不做娱乐动作：

| 动作 | 帧数 | 用途 |
|---|---:|---|
| idle 站立 | 4 | 空闲、待命 |
| walk 走路 | 4 | 在办公室移动 |
| sit 坐下 | 2 | 回到工位 |
| typing 敲键盘 | 4 | 正在执行任务 |
| thinking 思考 | 4 | 等待模型、规划中 |
| reading 查资料 | 4 | 知识库检索 |
| presenting 汇报 | 4 | 生成结果、会议展示 |
| error 卡住 | 2 | 工具失败、需要用户介入 |

第二批再扩展：

- meeting 开会
- handoff 交接任务
- search 翻资料柜
- export 递交文件
- sleep 低活跃待机

## 7. 状态映射

办公室里的角色动作必须尽量映射真实状态：

| Soul 状态 | 角色表现 |
|---|---|
| 无会话 / 无任务 | idle |
| 用户正在输入 | idle + 轻微等待动作 |
| LLM 请求中 | thinking |
| 工具调用中 | typing 或 reading |
| 知识库检索 | reading |
| 子分身协作 | walk to meeting / handoff |
| 产物生成完成 | presenting |
| 工具失败 / 超时 | error |

原则：

- 不编造 token、收益率、成功率等没有真实来源的数据。
- 可以用真实对话数、最近任务、工具调用、产物文件、错误状态驱动表现。
- UI 面板显示数据，角色负责表达状态。

## 8. 专家变体规则

所有专家都基于同一个小猪骨架变化，避免每个分身像不同产品。

可变区域：

- 马甲颜色
- 工牌图标
- 手持物
- 头部配件
- 脖挂式耳机颜色
- 屏幕内容
- 小包上的标记
- T 恤颜色或胸前小图案
- 外套 / 马甲颜色，仅作为专家变体叠加
- 休闲鞋点缀色

建议变体：

| 分身 | 变体元素 |
|---|---|
| 小堵-工商储专家 | 白色短袖 T、深蓝点缀、图纸卷、储能柜小图标 |
| 财务分析专家 | 绿色工牌、账本、计算器屏幕 |
| 法务专家 | 深灰外套、合同夹、印章图标 |
| 产品经理 | 橙色便签、路线图板、白板笔 |
| 设计大师 | 紫/青点缀、色卡、画板 |
| 项目经理 | 工装外套、进度板、甘特图屏幕 |
| 市场分析师 | 放大镜、趋势折线屏幕 |
| 电气工程师 | 蓝色外套、电气图纸、工具包 |

不建议：

- 每个专家换成不同动物。
- 服装差异过大导致主 IP 消失。
- 直接用真实行业制服堆满细节，小尺寸会糊。
- 通过西装、领带、皮鞋表达专业性；Soul 的专业感应该来自工具和状态，不来自正装。

## 9. 色彩标准

基础小猪：

- 皮肤：柔和粉，不要过艳。
- 鼻子：比脸略深一档。
- 轮廓：深棕或近黑，不能纯灰无边界。
- T 恤：白色、浅灰或低饱和浅色为主，便于专家色叠加。
- 外套 / 马甲：只在专家变体中少量叠加，不作为默认主视觉。
- 鞋：深色低帮休闲鞋，鞋底可用浅色像素边增强识别。
- 高光：少量暖白，不做大面积玻璃质感。

办公室整体：

- 可用暖灰、深蓝灰、木色少量点缀。
- 不继续使用纯白产品渲染图作为主视觉。
- 像素角色必须能从背景中跳出来。

## 11. 生产顺序

建议执行顺序：

1. 主形象 3/4 站立设定图。
2. `48x64` idle / walk / typing 三个最小动作。
3. `32x32` 头像。
4. 3 个专家变体：小堵、法务、设计大师。
5. 接入办公室运行状态。
6. 再做完整办公室地图。

不要先重做办公室大场景。先让角色活起来。

## 12. 下一步验收标准

第一版角色设定通过条件：

- 缩到 `32x32` 仍能看出是猪。
- 不看说明也知道是办公室工作角色。
- 和当前 Soul 像素风 UI 不冲突。
- 能自然扩展成不同专家分身。
- 站立、走路、敲键盘三个动作风格统一。
