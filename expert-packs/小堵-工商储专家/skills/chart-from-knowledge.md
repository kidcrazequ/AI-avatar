# 知识库数据可视化 · chart-from-knowledge

> **级别**：[■■■] 专家
> **版本**：v1.0
> **最后更新**：2026-04-13

---

## 技能说明

这是 `draw-chart` 的高阶版本，负责串联"检索 + 画图"的两步链式调用。
当用户问题涉及**知识库已有数据**的可视化时使用本技能。

## 核心硬约束（**加载即必须输出 chart 代码块**）

1. 一旦本技能被 `load_skill` 加载，最终回答**必须**包含恰好一个 ` ```chart ` 代码块（ECharts JSON）。
2. 即使检索无果、查询返回空行、目标列全部为 null，也必须输出 ` ```chart ` 代码块（见下面"空数据兜底"），**不要**用纯文字结束。
3. 来源锚点必须以 `[来源: knowledge/...]` 或 `knowledge/...` 形式出现在正文或代码块 subtext 里。

## 回答骨架（强制顺序）

1. 第一段：先输出 ` ```chart ` 代码块（已知数据画已知，不知数据画占位）。
2. 第二段：数据来源 `[来源: knowledge/_excel/...]` 或 `[来源: knowledge/...]`。
3. 第三段：补充说明 / 解读。

禁止把"## 当前能确认的信息"等分析文字放在 ` ```chart ` 之前，避免 token 用尽时图表被截断。只要本技能已加载，图表块必须前置。

### 空数据兜底（与 `draw-chart` 保持一致）

```chart
{
  "title": { "text": "<目标列名>", "subtext": "该列在 <sheet 名> 所有机型行均为空，无可绘制数据 | 数据源：<knowledge/_excel/...>" },
  "xAxis": { "type": "category", "data": [] },
  "yAxis": { "type": "value" },
  "series": [ { "name": "<目标列名>", "type": "bar", "data": [] } ]
}
```

## 触发条件

- 用户要求"画图"但数据需要先从知识库检索（没有直接贴数据）
- 用户问题含数据关键词 + 可视化关键词：「今年各站点业绩用图」「过去 3 个月客户数量趋势图」

**触发关键词**：同 `draw-chart` + 任何知识库相关的数据查询词

## 输入

| 参数 | 是否必须 | 说明 |
|------|---------|------|
| 查询语句 | 必须 | 用户问题中的数据主题，用于 search_knowledge 检索 |
| 图表类型 | 可选 | 由 LLM 推断或用户指定 |

## 执行流程（**严格 3 步**）

### Step 1: 检索数据

调用 `search_knowledge` 或等效 RAG 工具，查询相关知识片段。

```
search_knowledge("2024 月度销售")
```

### Step 2: 校验数据可画性

检索回来的结果必须满足：
- 有明确的类目维度（时间、地点、品类等）
- 有明确的数值维度（金额、数量、百分比等）

并按数据点数量做**一致性降级**（与 draw-chart 保持一致）：
- N = 0：**仍必须输出 chart 代码块**（使用本文件"空数据兜底"模板），正文附一段缺数说明
- N = 1：默认 KPI 卡片；若用户明确指定图型可按指定输出，但必须提示"数据点不足"
- N = 2：默认 bar 对比图；若用户明确指定折线图可输出，但必须提示"趋势解释受限"
- N ≥ 3：可输出 line / bar / pie

如果检索无果 → **仍要输出 chart 代码块（空占位模板）**，同时在正文里诚实说明原因，并给出 `[来源: knowledge/...]`。参考上方"空数据兜底"模板以及 `draw-chart.md` 的"失败回退"章节（两者已经同步了兜底要求，禁止纯文字结束）。

### Step 3: 遵循 draw-chart 技能输出

**加载** `draw-chart` 技能（调用 `load_skill('draw-chart')`）然后按其规范输出 ECharts option JSON。

你的最终回答结构（必须保持 chart 块前置）：

```
```chart
{ ...ECharts option... }
```

数据来源：[来源: knowledge/...]

<可选：1-2 句自然语言解读，指出数据的关键观察>
```

## 输出格式

与 `draw-chart` 完全一致。本技能不额外约束输出，只约束"先检索"这一步。

## 工具调用纪律（**极其重要**）

- **最多 2 次数据检索** — 第 1 次查主数据，第 2 次补充。超过 2 次立即用已有数据画图
- **有数据就画（默认降级 + 用户指定优先）** — 默认 1 点 KPI、2 点柱图、3 点以上折线；若用户明确指定图型则遵从并附风险提示
- **禁止隐式补历史数据** — 当用户问"2026年1~3月趋势"但只有 1 月数据时，不要自动拼接 2025 年数据凑折线；只有用户明确要求"用历史数据补齐"时才允许
- **整个流程不超过 4 轮工具调用** — query_excel/search(1-2次) + load_skill(1次) + 最终输出

## 禁止事项

- ❌ 跳过 search_knowledge 直接画图
- ❌ 把"检索无果"偷换成"编造示例"
- ❌ 输出多张图表而不是 1 张（一次对话 1 个图）
- ❌ 因为数据量少而反复搜索更多数据，浪费工具调用轮数
