## ⚠️ 技术栈说明（**关键 — 必读，输出前自检**）

本项目使用 **Apache ECharts 5+**，**不是** Chart.js / Plotly / Vega-Lite / D3 / Recharts / 任何其他图表库。

LLM 训练数据里 Chart.js 比 ECharts 流行很多，你可能下意识写出 Chart.js 格式，**这会直接导致渲染失败**（`Cannot create property 'series' on boolean 'true'` 之类的晦涩错误）。

### ❌ Chart.js 格式（错误示例 — 绝对不要输出这种）

```json
{
  "type": "line",
  "data": {
    "labels": ["2025年7月", "2025年8月", "2025年9月"],
    "datasets": [
      { "label": "效率", "data": [88.25, 88.28, 88.27] }
    ]
  }
}
```

### ✅ ECharts 格式（正确）

```json
{
  "title": { "text": "..." },
  "tooltip": { "trigger": "axis" },
  "xAxis": { "type": "category", "data": ["2025年7月", "2025年8月", "2025年9月"] },
  "yAxis": { "type": "value" },
  "series": [
    { "type": "line", "name": "效率", "data": [88.25, 88.28, 88.27] }
  ]
}
```

### 关键差异对照表

| Chart.js（错） | ECharts（对） |
|---|---|
| 顶层 `"type": "line"` | 顶层**没有** type；type 在 `series[].type` |
| `data.labels` | `xAxis.data` |
| `data.datasets[]` | `series[]` |
| `datasets[].label` | `series[].name` |
| `datasets[].data` | `series[].data` |
| `options.plugins.title.text` | `title.text` |
| `options.scales.y.beginAtZero` | `yAxis.type: "value"`（默认从 0）|

### 输出前自检清单（必查）

✅ 顶层有 `xAxis`、`yAxis`、`series` 这三个字段（饼图除外，饼图只有 `series`）
❌ 顶层**没有** `type`、`data.labels`、`data.datasets`、`options.plugins`
❌ `series` 不是 `datasets`
❌ `xAxis.data` 不是 `data.labels`

如果你不确定格式，对照下面"输出格式硬约束"段的示例 1 / 2 / 3 逐字段检查。

## 技能说明

当用户的问题涉及数据可视化时，用 ECharts 输出一份高级感图表，直接内联在回答中渲染。
这个技能只负责"画图"，不负责"找数据"——找数据是 `search_knowledge` 或 `chart-from-knowledge` 的职责。

## 触发条件

在以下场景应读取并遵循本技能：

- 用户显式要求：「用图表 / 画个图 / 可视化 / 做一张趋势图 / 做饼图 / 柱状图对比」
- 用户的问题天然适合图表表达：月度/季度/年度趋势、占比分布、同比/环比对比、KPI 看板

**触发关键词**：`图表` `可视化` `趋势` `对比` `占比` `分布` `同比` `环比` `柱状` `折线` `饼图` `画图`

## 输入

| 参数 | 是否必须 | 说明 | 示例 |
|------|---------|------|------|
| 数据 | 必须 | 要可视化的数据；优先从已检索的上下文取，否则先 `search_knowledge` 再用 | 月度销售额 |
| 图表类型 | 可选 | **用户指定时必须严格遵从**（说"折线图"就必须用 line，说"柱状图"就必须用 bar）；未指定时按数据性质选（趋势→line，占比→pie，对比→bar） | 折线 / 柱状 / 饼 |
| 标题 | 必须 | 你必须生成一个含义明确的标题 | "2024 年月度销售额" |

## 数据来源规则（**硬约束**）

按优先级：

1. **Excel / CSV 数据 → 必须用 `query_excel`** — 看 system prompt 顶部"可查询 Excel 数据源"列表，找到对应的 file 和 sheet，用 filter 精确过滤行。**禁止**用 search_knowledge 查 Excel 数据（模糊匹配会丢行）。示例：用户问"215 机型 2026 年 1~3 月设备侧效率"：
   ```
   query_excel({
     file: "00_工商储-产品质量指标dashboard_260303",
     sheet: "月度",
     filter: {
       "机型": "215",
       "月份": {"$gte": "2026-01", "$lte": "2026-03"}
     },
     columns: ["月份", "设备侧效率"]
   })
   ```
2. **非 Excel 的知识库数据** → 用 `search_knowledge` 模糊检索
3. 若用户在消息里直接贴了数据 → 使用用户数据
4. 若以上都没有 → **不要编造**。诚实说明缺什么数据，请求用户提供或指出 Excel 文件

**严禁行为**：
- ❌ 凭记忆/推测填入数字（模型训练数据不是事实）
- ❌ 用「大约」「假设」「估计」等词 + 编造的数值
- ❌ 用占位数据画出"示意图"——即使用户没明说，也要等真实数据
- ❌ **对 Excel 用 search_knowledge** 而不是 query_excel

### 参数模板（**格式化问题必须逐字照抄**）

当用户问题匹配下列格式时，**禁止自由发挥**，必须按给定模板填槽调用 `query_excel`，否则同问多次会得到不同参数 → 查到不同数据 → 图漂：

**格式 A：`N 机型 YYYY 年 M1~M2 月 X 指标`**（例："215 机型 2026 年 1~3 月设备侧效率折线图"）

```
query_excel({
  "file": "00_工商储-产品质量指标dashboard_260303",
  "sheet": "月度",
  "filter": {
    "机型": "<N>",
    "月份": {"$gte": "<YYYY>-<MM1>", "$lte": "<YYYY>-<MM2>"}
  },
  "columns": ["月份", "<X 指标中文原名>"]
})
```

**填槽硬规矩**（每一条违反都会破坏 cache 命中）：
- `<N>` 取用户原话里的机型数字（`215`/`372` 等），**不加单位、不加机型两字、不加引号外再套引号**
- `<YYYY>-<MM>` 用零填充两位月份（`2026-01` 而非 `2026-1`）
- `filter.月份` 永远用 `{"$gte": ..., "$lte": ...}` 对象形式；单月也用 `"$gte"=="$lte"`，**不要降级成字符串 `"2026-01"`**
- `columns` 永远两列 `["月份", "<指标>"]`，**不要额外追加"机型""容量"等未被问到的列**
- `"sheet"` 固定写 `"月度"`；季度/年度问题走不同模板，不要混用

**格式 B：`N 机型 YYYY 年 X 指标`**（没给月区间，按全年）

```
query_excel({
  "file": "00_工商储-产品质量指标dashboard_260303",
  "sheet": "月度",
  "filter": {
    "机型": "<N>",
    "月份": {"$gte": "<YYYY>-01", "$lte": "<YYYY>-12"}
  },
  "columns": ["月份", "<X 指标中文原名>"]
})
```

**以上两条模板的存在目的**：让"N 机型 YYYY 年 M~M 月 X 指标"这类问题每次跑出字符一致的 tool 参数，配合 chatStore 的 `normalizeQueryExcelArgs` cache 达到"同问同答"。偏离模板的写法本身合法但会破坏缓存命中，**仅当用户问题不符合格式 A/B 时**才允许自由构造参数。

## 数据完整性守护（**画图前必检**）

**画图前的强制检查清单**，违反任何一条都会产出丑图（实测案例：单点折线图 + markLine 拉横线 + 空 X 轴刻度）。

### 1. 检查数据点数量

`query_excel` 返回 N 行后，**先数行数**：

| 数据点数 | 默认推荐图型 | 说明 |
|---|---|---|
| **N = 0** | 不画图，输出"无数据"文字提示 | 无可视化数据 |
| **N = 1** | KPI 卡片（标题大字 + 单值 + 副标说明数据范围） | 若用户明确指定图型（如“必须折线图”），可按指定输出，但必须附“数据点不足”提示 |
| **N = 2** | bar 对比图（2 个柱子 + 涨跌幅标签） | 若用户明确指定折线图，可输出折线并提示趋势解释受限 |
| **N ≥ 3** | line / bar / pie | 正常按用户意图或数据语义选型 |

**铁律**：折线图至少需要 **3 个时间点**才能形成"趋势"概念。1-2 个点画成折线图视觉上等于一根斜线或一个孤点，毫无信息量。
**一致性约束**：当用户问"X 月到 Y 月趋势/折线图"但实际数据点不足 3 个时，默认按上表推荐图型；如果用户明确指定图型则遵从用户要求，但必须在正文中添加固定提示：`当前数据点不足，图形仅反映已有月份，趋势解释受限。`

### 2. 数据稀疏必须诚实告知

如果 `query_excel` 返回的行数**远少于用户预期**（比如用户问"2026 年 1-3 月趋势"但只返回 1 行 = 只有 1 月有数据），**必须在文字部分明确说明**：

> "Excel 数据源中 215 机型仅有 2026 年 1 月（统计周期 2601）的设备侧效率数据，2-3 月暂未录入。当前图形仅反映已录入月份，趋势解释受限。"

默认切换为 KPI（N=1）或柱图（N=2）。仅当用户明确要求“可用历史数据补齐”时，才允许补历史数据。**禁止无提示地硬画一个"看起来像 1-3 月"但实际只有 1 月数据的折线图**。

### 3. markLine / markPoint 的数据点门槛

| 标注类型 | 启用条件 |
|---|---|
| `markPoint: { data: [{ type: 'max' }, { type: 'min' }] }` | 数据点 ≥ 3 |
| `markLine: { data: [{ type: 'average' }] }` | 数据点 ≥ 3 |
| `markLine: { data: [{ yAxis: <某固定阈值> }] }` | 任意（基线/目标线）|

**禁止反例**：单点数据 + `markLine.type: 'average'` → average 退化为水平线 → ECharts 会把箭头拉到图表右边缘，标签溢出截断（实测见 v0.6.0 之前的 215 机型截图）。

### 4. X 轴不要补空刻度

如果数据只到 2026 年 1 月，X 轴 **只显示 1 月**，**不要**写 `xAxis.data: ['2026年1月', '2026年2月', '2026年3月']` 把后面两个月作为占位。空刻度让图表看起来像"数据丢失"，比"X 轴只有 1 个点"更糟。

正确：用真实数据范围生成 X 轴；ECharts 会自动处理稀疏 category。

## 执行流程

1. **确认数据来源** — 上下文有数据 / 调用检索 / 用户已提供 —— 三选一
2. **选择图表类型** — 按数据性质：趋势→line / 占比→pie / 对比→bar / 分布→scatter
3. **构造 ECharts option** — 使用下面的"输出格式硬约束"
4. **输出** — 在回答中先写一两句解读（"数据显示 X 月销售额最高..."），然后立即紧跟一个 `chart` 代码块
5. **结束** — 不要在图表后追加"希望这对您有帮助"等套话

### 工具调用纪律（**极其重要**）

- **最多 2 次 query_excel** — 第 1 次查主数据，第 2 次补充（如查对比期）。超过 2 次说明你的 filter 写错了，停下来用已有数据画图
- **有数据就画（默认降级 + 用户指定优先）** — 默认 1 点 KPI、2 点柱图、3 点以上折线；如果用户明确指定图型则遵从，并附“数据点不足、趋势解释受限”提示。**不要因为数据少继续搜索更多数据**，用户要的是稳定输出，不是无限补数策略
- **load_skill 只调 1 次** — 加载完 draw-chart 技能后立即画图，不要再加载其他技能
- **整个流程不超过 4 轮工具调用** — query_excel(1-2次) + load_skill(1次) + 最终输出 = 最多 4 轮。如果你发现自己在第 5 轮还没画图，**立即用已有数据输出 chart**

## 输出格式硬约束

必须输出**恰好一个** fenced code block，语言标签是 `chart`，内容是合法的 ECharts option JSON。

```chart
{
  "title": { "text": "标题", "subtext": "副标题" },
  "tooltip": { "trigger": "axis" },
  "legend": { "data": ["系列名"] },
  "xAxis": { "type": "category", "data": ["1月", "2月", "3月"] },
  "yAxis": { "type": "value" },
  "series": [
    { "name": "系列名", "type": "line", "data": [120, 132, 101] }
  ]
}
```

**代码块内禁止的内容**：
- 非 JSON（注释、尾逗号、单引号、变量、**JavaScript 函数**）
- **严禁 `function` 关键字**：`formatter`、`valueFormatter`、`color` 等字段不得使用 `function()`。只能用 ECharts 的字符串模板（如 `"{b}: {c}"`）或直接省略
- 解释文字（必须放在代码块**之前**作为普通 markdown 段落）
- 空对象或空 series 数组

## UED 设计规范（摘自 ued-expert.md）

### ✅ 必须遵守

1. **禁止 3D** — `series[].type` 不得是 `bar3D` / `line3D` / `surface` / `scatter3D`
2. **Y 轴从 0 开始（一般情况）** — `yAxis` 不写 `min` 字段（默认 0）。**例外**：见下面"百分比/效率类数据" 规则。
3. **百分比/效率类数据自适应 Y 轴范围**（v0.6.3 新）— 如果数据是 % 类（设备效率、转化率、SOH、合格率等），数据值通常都集中在 80-95% 区间，**强制用 0-100 会让 4-5pp 的差异看起来像直线**。规则：
   - 计算数据 `[min, max]`
   - 设 `yAxis.min: floor((min - 2) / 5) * 5`（向下取整到最近 5 的倍数，留 2 个百分点 buffer）
   - 设 `yAxis.max: ceil((max + 2) / 5) * 5`（向上取整到最近 5 的倍数，留 2 个百分点 buffer）
   - 设 `yAxis.axisLabel: { formatter: "{value}%" }` 加百分号
   - 例：数据 88.25-90.6% → `yAxis.min: 85, max: 95`，差异清晰可见
   - 这是 UED "禁止截断 Y 轴" 的合理例外：百分比本身就是相对量，不会误导
4. **标题 + 副标题必填**：
   - `title.text` 简洁（**≤ 20 字符**），不要把数据范围塞进 text 里
   - `title.subtext` 必须有，写"数据来源 / 时间范围 / 项目数 / 单位"
   - **错误反例**：`text: "215机型设备侧效率趋势图（2025年7月-12月）"`（28 字符，会和 legend 撞车）
   - **正确**：`text: "215 机型设备侧效率趋势"`、`subtext: "数据源：xxx · 2025-07 至 2025-12 · 单位 %"`
5. **系列数 ≤ 5** — 主题 palette 只提供 5 色，超过会重复
6. **🚫 严禁硬编码颜色** — 不写任何 `color` 字段，让 pixel 主题自动接管。这是**最容易被忽略**的规则，实测 215 机型截图：LLM 输出 `series: [{ type: 'line', itemStyle: { color: '#5470C6' } }]` 把折线染成默认蓝，和 LED 粉主题完全脱节。**严禁出现以下任意一种**：
   - ❌ `color: ['#xxx', '#yyy']`（顶层）
   - ❌ `series: [{ ..., color: '#xxx' }]`
   - ❌ `series: [{ ..., itemStyle: { color: '#xxx' } }]`
   - ❌ `series: [{ ..., lineStyle: { color: '#xxx' } }]`
   - ❌ 任何 `'#5470C6'`、`'#91CC75'`、`'#5B7EE8'` 等"通用 ECharts 蓝/绿"色值
   - ✅ 留空，让主题 palette 按顺序分配（粉 → 绿 → 蓝 → 黄 → 紫）
   - **即使被剥掉**：渲染器在 v0.6.16 起会主动剥掉所有显式 series.color，但你应该从 prompt 阶段就不写，避免误导用户阅读 JSON
7. **X 轴类目 > 12 时改用横向** — 如月份可以竖着，但站点、SKU 这种多条目必须用横向柱状图（`yAxis.type: 'category'`, `xAxis.type: 'value'`, `series[].type: 'bar'`）
8. **单值 KPI 必须带对比** — 如果只有一个数字，必须同时展示"上期/去年同期"作为第二个系列或副标题

### ✅ 高级视觉（主题已内置，你只需配合）

主题已内置以下视觉效果，**不需要你在 JSON 里手动写**：
- 柱状图自动圆角顶部
- Tooltip 自动毛玻璃效果
- 配色自动 5 色暗底优化
- legend 自动放底部居中（v0.6.3 起，避免和长标题撞车）
- aria 启用但不渲染 decal pattern（v0.6.3 起，色盲友好但不污染视觉）

**你需要配合的**：
1. **折线图少写 symbol** — 不要写 `showSymbol: true`，主题默认 hover 才显示点，更干净
2. **areaStyle 默认关闭**（v0.6.3 起）—— 主题不再默认给折线图加渐变 areaStyle（之前多 series 折线图会重叠变浑浊）。**单 series** 折线图想要渐变效果，可在 `series[0].areaStyle: {}` 里显式写空对象（主题会自动注入渐变色）；**多 series 严禁** `areaStyle`
3. **饼图用环形** — `radius: ["45%", "70%"]`，比实心饼更现代
4. **markPoint 标注极值** — 趋势图（数据点 ≥ 3 时）加 `markPoint: { data: [{ type: "max" }, { type: "min" }] }` 突出关键数据
5. **grid 留白** — 不要写 grid 字段，主题默认 `{top: 64, bottom: 56, left: 56, right: 64}` 已为标题、底部 legend、markLine 标签留够空间

### ❌ 严禁

- 3D 图表（反模式）
- 彩虹调色板（5 色以上）
- 截断 Y 轴
- 超过 12 个扇区的饼图（改用横向柱状图 + Top 10）
- 在图表内重复标题文字（tooltip / label 已经能表达）
- 手动写 `areaStyle`、`itemStyle.borderRadius`、`tooltip` 样式（**主题已接管**，手写会覆盖）
- **手动写 `legend.right` / `legend.top` / `legend.left` / `grid.left` / `grid.top` / `grid.right` / `grid.bottom`** —— 主题已经精确计算了位置（top: 12 / 72 等），手动覆盖会让 legend 撞上副标题、grid 把数据挤出图表
- **legend.icon 用 emoji 字符或符号字符**（⛑️ 🔵 ▲ 之类）—— legend.icon 只能用 ECharts 内置形状字符串：`'roundRect' / 'circle' / 'rect' / 'triangle' / 'line' / 'pin' / 'arrow' / 'none'`。**最好不写**，让主题统一接管为 `roundRect`
- **series[].name 字段含 emoji / 表情符号** —— 部分 ECharts 渲染器会把 series.name 当作 legend label 一起渲染，emoji 会污染图例
- **写 `markLine` / `markPoint` 不检查数据点数量** —— 见上文"数据完整性守护 §3"

## 示例

### 示例 1：月度趋势（line）

**用户问**：用图表展示 2024 年各月销售额

**检索**：从 `月度销售.xlsx` sheet `2024` 得到 12 条数据

**你的回答**：

2024 年销售趋势显示上半年增长平稳，6 月达到年度峰值，下半年略有回落。全年总销售额 1,485 万元。

```chart
{
  "title": { "text": "2024 年月度销售额", "subtext": "单位：万元 | 数据来源：月度销售.xlsx" },
  "tooltip": { "trigger": "axis" },
  "xAxis": {
```
